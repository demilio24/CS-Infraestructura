## 1. What this is

A public-facing website for **Nils Digital** that serves as the **SEO front door + warm-up + routing layer** for the two existing VSL funnels:

- Marketing system → `/Nils/funnel/vsl.html`
- Automation audit → `/Nils/funnel/automation-vsl-funnel-direct.html`

The website is **NOT a sales funnel**. The VSL funnels do the converting. The website's job is to capture organic search traffic via the blog, build trust, and route visitors to the right VSL.

**Buyer this is for:** Owners of $1M+ local service businesses (same persona for both offers).

## 3. Design system

### Tokens (in `assets/css/tokens.css`)
- **Container max:** `1400px` (was 1200 — bumped per user feedback so wide screens fill better)
- **Type:** Instrument Serif (headlines), Instrument Sans (CTAs / labels / nav), Google Sans + DM Sans (body)
- **Colors:**
  - Marketing accent: `#046bd2` (blue, from Marketing VSL)
  - Automation accent: `#0fbf7a` / `#087a4d` (green, from Automation VSL)
  - Trustpilot brand green: `#00b67a` (used only inside the Reviews wall — intentional brand reference, not a Nils token)
- **Headline formula** (locked, baked into all section heroes):
  - Eyebrow (Instrument Sans pill, blue, uppercase, letter-spaced)
  - H1 / H2 (Instrument Serif, italic accent words encoded by offer color — blue for Marketing, green for Automation)
  - Lead `<p class="lead">` (DM Sans, ink-soft color)
- **Copy rules** (enforce on every edit):
  - **No em-dashes** in user copy (`—` / `&mdash;`). Use comma, period, colon, parens. En-dashes for ranges (`3–6`) OK.
  - **No `text-align: justify`** on body copy.
  - **One H1 per page** — section heroes use H2 (proof / about / contact). Only the top hero is H1.
  - **Token discipline** — every CSS color or sizing value should reference a `var(--token)`. No hardcoded hex except the Trustpilot brand green (commented as such where used).

### B3 brutalist button (route-half CTA, the default everywhere `.route-halves` is used)
White pill, brand-colored text (blue for marketing card, green for automation card), 4px hard offset shadow at rest, presses to 2px shadow + `translate(2px, 2px)` on hover. Defined in `components.css` under `.route-half .btn`.

## 5. Blog stack

### Publishing flow (BabyLoveGrowth)
1. BabyLoveGrowth writes a raw post HTML to `blog/posts/<slug>.html`. The post should include `<meta name="blg-tag" content="marketing|automation">` (and ideally `blg-date`, `blg-read-time`, `blg-subtitle` meta tags).
2. Run the post-process script:
   ```bash
   node Nils/website/scripts/inject-post-cta.mjs Nils/website/blog/posts/<slug>.html
   ```
   This wraps the post body in `blog/_template.html` and injects the right inline CTA based on the post's primary tag (Marketing-tagged posts get a blue CTA to the Marketing VSL; Automation-tagged posts get a green CTA to the Automation VSL).
3. Rebuild the Pagefind search index:
   ```bash
   cd Nils/website && ./scripts/build-search.sh
   ```
4. Commit and push. GitHub Pages redeploys.

### Search
Pagefind generates a static search index at `Nils/website/pagefind/` (gitignored — rebuild locally before deploying). The blog index page has a search input wired to Pagefind UI; `⌘K` / `Ctrl+K` focuses it.

### Adding a new tag
Edit two places:
1. The toolbar `<button class="blog-tag">` list in `blog/index.html` (add a new pill).
2. The `CTA_BY_TAG` object in `scripts/inject-post-cta.mjs` (add the new tag's headline + body + button + href + cls — `cls` controls the CTA gradient color, use `marketing` or `automation`).

## 6. SEO

- **Sitemap:** `sitemap.xml` (2 URLs — home + `/blog/`). Subpage redirects are `noindex`. As blog posts get published, append them to the sitemap manually (or write a script).
- **robots.txt:** Allows root, disallows `_drafts/`, `scripts/`, `pagefind/`.
- **Organization JSON-LD:** Inlined in `<head>` of `index.html` (Nils Digital, founder Emilio, areaServed US, knowsAbout keywords).
- **Article JSON-LD:** In `blog/_template.html`, populated per-post by `inject-post-cta.mjs`.
- **Canonical:** `https://nilsdigital.com/Nils/website/` is set as the canonical URL across all pages, anticipating the eventual DNS migration. Currently the actual serving URL is the demilio24.github.io path — search engines should still consolidate them via the canonical tag.

## 7. Open threads

### DNS / custom domain (action item)
`nilsdigital.com` currently 301s into GHL-hosted pages (verified — apex hits `nilsdigital.com/home-104085` which is a GHL Funnels-built page). To move the website to the apex:

1. **In GoHighLevel:** unlink `nilsdigital.com` from the agency's custom-domain settings (or move it from GHL to GitHub Pages).
2. **DNS provider (wherever the registrar is):** Update A/AAAA records to point at GitHub Pages' edges (`185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`) — or CNAME to `demilio24.github.io` if using a subdomain.
3. **GitHub Pages settings** for the `Websites` repo: set `nilsdigital.com` as the custom domain (this adds a `CNAME` file to the repo root). Wait for Let's Encrypt to provision the cert (usually 5-15 min).
4. **Once live:** the site will serve at `https://nilsdigital.com/Nils/website/`. If you want it at apex root (`https://nilsdigital.com/` → site directly), either move all the website files to the repo root or set up a root-level redirect.

### Microsoft Clarity (deferred at launch per Task 0 decision)
A commented `<script>` block lives in `assets/partials/head-meta.html` with `CLARITY_ID` as a placeholder. When you register a Clarity project at `clarity.microsoft.com`:
1. Uncomment the Clarity block in `head-meta.html`.
2. Replace `CLARITY_ID` with the real ID.
3. The `<head>` block in `index.html` is inlined (not pulling from the partial at runtime), so you'll also need to paste the updated Clarity block into `index.html` and `blog/index.html` directly.
4. Update `~/.claude/projects/.../memory/reference_clarity_tracking_ids.md` with the new ID.

## 8. How to make common changes

| Want to... | Do this |
|---|---|
| Update the hero headline | Edit the `<section id="top">` block in `index.html`. Locked V1 copy is: eyebrow "For local service businesses" / H1 "If you want to get more clients and automate your busywork, you're in the right place." / lead about Google Ads + automations. |
| Swap a testimonial card photo | Edit the `<img src="...">` in the relevant `<article class="case">` block in `index.html`. Pull a new URL from GHL via the same path pattern. |
| Add a new Trustpilot review to the reviews wall | Edit the `<div class="reviews-wall">` block in `index.html`, add an `<img src="..." alt="...">` to the `.grid`. Grid auto-fits. |
| Change the route-half buttons | Edit `.route-half .btn` and `.route-half:hover .btn` in `components.css`. The B3 style propagates to all pages using `.route-halves`. |
| Change the container width | Edit `--container-max` in `tokens.css`. Currently 1400px. |
| Add a new section | Add markup to `index.html` between existing sections. Use `<section class="anim">` for scroll-reveal. Add `id="..."` if you want a nav anchor. If the section needs styling, add a new CSS file under `assets/css/` and link it in the `<head>` of `index.html`. |
| Change a route-half copy or destination | Edit the `<a class="route-half marketing/automation">` blocks in the `#offers` AND `.contact-routes-echo` sections of `index.html`. Both render the same dual-CTA pattern. |
| Update Emilio's portrait | Replace the `<img class="portrait" src="...">` URL in the `.about-hero.v2 .inner` block. |
