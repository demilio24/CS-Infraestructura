# Systema Floyd — Sheets Snapshot

Apps Script that reads the four canonical Systema Floyd registration sheets, aggregates them into a JSON snapshot, and writes that snapshot directly to GitHub. Powers the dashboard at `Tom_Systema_Floyd/dashboard/index.html` (which loads `snapshot.json` at page-load time and renders it).

**Owner:** Tom Floyd (Systema Floyd Martial Arts)
**Built by:** Emilio (Nils Digital)
**State:** Live. Triggers on a 5-min cadence under `emilio@nilsdigital.com`.

---

## What this replaced

Prior to 2026-05-08, refreshes were driven by `.github/workflows/refresh-systema-snapshot.yml` — a cron-scheduled GitHub Actions workflow that called the script's web app via Python and committed the result. Configured every 5 min, but in practice fired roughly **once per hour** under GitHub's free-tier scheduled-workflow throttling. End-to-end staleness was up to 2 hours on bad days.

The Apps Script time-driven trigger fires on schedule reliably (verified end-to-end same day). End-to-end staleness is now ≤ 5 min.

The old workflow's `schedule:` trigger has been removed; `workflow_dispatch:` stays as a manual rebuild button.

## Outputs (what gets written to GitHub on each run)

| Path | Purpose |
|---|---|
| `Tom_Systema_Floyd/dashboard/snapshot.json` | Live snapshot consumed by `index.html`. Overwritten every run. |
| `Tom_Systema_Floyd/dashboard/history/<UTC-date>.json` | Daily archive. Same content as snapshot.json, written under that day's UTC date. Lets you reconstruct historical roster state. |
| `Tom_Systema_Floyd/dashboard/history/index.json` | List of all archived dates. Refreshed every run from the actual directory contents (so deleted archives self-correct). |

Each run produces **3 separate commits** (one per file via separate Contents API PUTs), authored as `github-actions[bot]` to match the pre-existing convention. The 3-commit signature 1 second apart is how to tell Apps Script-driven runs apart from any future workflow-driven runs.

## How to develop locally

```
cd Tom_Systema_Floyd/sheets-snapshot/apps-script
npx --yes @google/clasp@latest push -f      # push local edits to Apps Script
```

clasp auth is shared with the billing dashboard project (same `~/.clasprc.json`, same `systemafloydsheets@gmail.com` account). Logging out of one logs out of the other.

To verify a code change end-to-end **without waiting for the 5-min trigger**:
1. Sign the Apps Script editor in as `emilio@nilsdigital.com`
2. Function dropdown → `testSnapshotPush` → Run
3. Read the Execution log

The function logs every step (`[1] effective user`, `[2] PAT present`, `[3] running pushSnapshotToGitHub`, then per-file write logs). Failures fall into a few buckets:

| Symptom | Likely cause | Fix |
|---|---|---|
| `[2] PAT err: GITHUB_PAT not set` | Property missing or empty | Set in Script Editor → gear icon → Script Properties |
| `HTTP 401: Bad credentials` | PAT expired or revoked | Generate a new fine-grained PAT (same permissions as before), update `GITHUB_PAT` |
| `HTTP 403: Resource not accessible` | PAT missing Contents:Write permission | Edit PAT at https://github.com/settings/personal-access-tokens, set Contents to "Read and write" |
| `HTTP 404: GET sheet ...` (during `buildSnapshot`) | A registration sheet wasn't shared with emilio | Re-share with `emilio@nilsdigital.com` as Viewer |
| `Service invoked too many times for one day: urlfetch` (during a manual editor run, NOT the trigger) | The editor was signed in as `systemafloydsheets@gmail.com` whose 20K quota is depleted | Sign editor in as `emilio@nilsdigital.com` and re-run. The autonomous trigger always runs as emilio and is unaffected. |

## Why the trigger is owned by emilio (not systemafloydsheets)

The 4 underlying sheets are owned by `systemafloydsheets@gmail.com`. That account has a **20,000/day** UrlFetch quota (consumer Gmail tier), which is shared across every Apps Script project that account runs. The billing dashboard project blew through this quota on 2026-05-08, taking the snapshot path with it.

`emilio@nilsdigital.com` is a Google Workspace account with a **100,000/day** quota (5×). Same per-run cost, much more headroom. The snapshot script's UrlFetch volume on its own is trivial (~1,150/day), but consolidating both Tom Systema Floyd Apps Script projects under emilio means a single shared quota counter that's effectively unhittable.

The four sheets are still owned by Tom's account — only **read** access was shared with emilio. No risk of accidental data modification.

## How to disable / pause the trigger

Open the Apps Script editor as `emilio@nilsdigital.com`:
- https://script.google.com/d/1EcPTHTRypJX_ywQXqj_LQuJMNk2RAjzSkIxsG8QzLe64jQtRXvEf6f8Y/triggers
- Hover the `pushSnapshotToGitHub` row → three-dot menu → Delete trigger

While disabled, refresh manually via the GitHub Actions workflow_dispatch button at https://github.com/demilio24/Websites/actions/workflows/refresh-systema-snapshot.yml (the cron schedule is gone but workflow_dispatch is intentionally still wired up for this exact case).

To re-enable: editor → function dropdown → `installSnapshotPushTrigger` → Run.
