¿Qué es esta base de datos?

Cada vez que se ejecuta un análisis de riesgo AML en n8n, el sistema consulta la API de RiskTech (AMLRISK) y la respuesta completa se guarda automáticamente en Supabase. Esta página permite consultar el historial de cada persona, ver en qué listas apareció y exportar los datos para auditoría.

La API devuelve un campo datos_listas con los códigos de las listas donde se consultó la persona, y un campo datos_tsti con los detalles de las coincidencias reales (cuando existen). El parámetro block indica si se encontró al menos una coincidencia.

Aclaración importante (correo de Harold Marín, RiskTech, junio 2025): Los números en datos_listas son los códigos de las listas donde se buscó a la persona — no son coincidencias. Solo cuando hay datos en datos_tsti (o block = true) la persona realmente coincide con una lista.

Etiquetas de listas

RiskTech agrupa las listas con etiquetas según su naturaleza:

PEPS Personas Expuestas Políticamente. Cargos públicos, funcionarios, jueces, fiscales.
SANCIONADOS Personas/entidades sancionadas por organismos como OFAC, ONU, Europol, ministerios públicos.
INFORMATIVAS Filtraciones (Panama Papers, Bahamas Leaks), procuradurías, paraísos fiscales.
NEWS Noticias relacionadas con AML (Aml_News Colombia, Panamá, Venezuela, etc).

Clasificación de riesgo de las listas

Clasificación validada con Keila Mulero (Savvy Compliance) en correo del 7 sept 2025. Nota especial: para Panamá (país que se evalúa), Keila pidió ampliar diligencia incluso para coincidencias en listas NEWS, ya que cualquier información local es relevante para el riesgo de blanqueo y financiación del terrorismo.

🔴 Alto Riesgo (Vinculantes / Críticas)

Coincidencia en estas listas requiere diligencia ampliada inmediata.

🔵 No Vinculantes (Informativas / Periodísticas)

Estas listas son informativas. Evaluar contexto antes de bloquear, pero documentar siempre.

Estructura de la respuesta de la API: Cada coincidencia en datos_tsti incluye los campos: lista (código), etiqueta (PEPS/Sancionados/Informativas/News con su color), nombre_apellido del registro coincidente, categoria (Fiscales, Narcotráfico, Paraíso Fiscal, etc), pais, detalle (descripción), y url con la fuente.
